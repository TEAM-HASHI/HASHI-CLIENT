# Final Report Checklist

최종 응답이나 PR 설명 전에 작업 결과를 빠짐없이 정리합니다.

## Include

- Jira key
- 변경한 주요 영역
- `docs` 영향 여부
- `.agents` 영향 여부
- 실행한 검증 명령과 결과
- 실행하지 못한 검증과 이유
- 남은 위험이나 후속 티켓 후보

## Avoid

- 확인하지 않은 내용을 완료처럼 말하지 않습니다.
- 실패한 검증을 생략하지 않습니다.
- 문서 변경이 필요 없다고 판단했다면 그 이유를 짧게 남깁니다.
- unrelated 변경을 함께 처리했다고 말하지 않습니다.

## Format

```markdown
작업 완료.

- Jira: HASHI-00
- 변경: ...
- 문서 영향: ...
- 검증: ...
- 남은 항목: ...
```
